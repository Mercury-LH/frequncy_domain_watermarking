# 隐形水印工坊 —— 频域水印在线体验 设计文档

日期：2026-07-04
状态：已与用户逐节确认

## 1. 目标与定位

把本仓库的频域水印研究项目（DFT / DCT / DWT 算法 + 攻击与指标体系）转化为一个**免费、点开链接即用**的 Web 应用。访客可上传图片并自定义水印（文字或图片），完成嵌入 → 提取 → 攻击实验的完整闭环。

定位（详见根目录 PRODUCT.md）：**展示优先，工具其次**（brand register）。品牌人格：学术、克制、可信。视觉水准对标北美一线科技公司官网（Stripe 式白底克制 × 学术出版物气质）。

### 已确认的需求决策

| 决策点 | 结论 |
|---|---|
| 功能范围 | 嵌入 + 提取 + 攻击演示（完整体验） |
| 自定义水印 | 文字输入（后端渲染位图，支持中文）和图片上传都支持 |
| 界面语言 | 中英双语切换，默认中文 |
| 访问范围 | 不要求中国大陆直连 |
| 部署方案 | Hugging Face Spaces（Docker，永久免费档） |
| 视觉风格 | 明亮学术纸感（真灰白底，非奶油色）+ 墨绿主色 + 衬线标题 |
| 页面结构 | 滚动叙事（scrollytelling）+ 页内工具区 |
| 无障碍 | WCAG AA + prefers-reduced-motion 适配 |

### 曾考虑并否决的方案

- **Vercel 单仓库（Next.js + Python serverless）**：加载最快，但 250MB 依赖体积限制紧，部署踩坑风险高。
- **纯前端 WASM（Pyodide / TS 重写）**：永不休眠，但与"需要有前后端"的要求矛盾，且首载 ~20MB 或需重写算法。

## 2. 产品体验设计

### 2.1 滚动叙事：「一张图的隐秘旅程」

页面主体为滚动驱动演示：右侧画面钉住（pinned），左侧短文案推进，共 8 幕：

| 幕 | 左侧文案要点 | 右侧画面 |
|---|---|---|
| 0 | Hero：「把水印藏进图像的频率里」+ 向下提示 | 静谧频谱纹理 |
| 1 | 这是一张普通的照片 | 示例照片淡入 |
| 2 | 图像其实是波的叠加——换个角度看它 | 照片过渡为 FFT 对数幅度频谱 |
| 3 | 水印写进中频系数——太低伤画质，太高易丢失 | 中频环带高亮，水印 bit 点亮 |
| 4 | 变回像素世界——肉眼完全看不出 | 原图 vs 含水印图并排；PSNR 数字滚动；×20 差值图 |
| 5 | 攻击它：JPEG 压缩、缩放、加噪 | 图像出现压缩伪影/噪点 |
| 6 | 水印依然在 | 提取水印浮现，NC 数值呈现 |
| 7 | 「轮到你了」 | 整屏墨绿 Drenched 过渡段，滑入工具区 |

实现决策：

- 演示帧**全部离线预生成**（`webapp/scripts/gen_story_assets.py` 调用现有算法产出素材，打包进前端）。滚动只做图像切换与数字动画，不做实时计算，保证 60fps。
- 滚动引擎：GSAP ScrollTrigger。
- 手机端：退化为纵向分段静态图叙事。
- `prefers-reduced-motion`：直接呈现静态分节（无 pinned 动画）。

### 2.2 工具区（同页，锚点可直达）

三个标签页：

1. **嵌入**：拖入图片（或一键示例图）→ 水印二选一（文字 / logo 图）→ 算法三选一（默认推荐 DCT，卡片附一句话说明：DFT 非盲需原图、DCT 盲提取、DWT 多分辨率）→ 强度滑杆（含默认值）→ 结果面板：含水印图、PSNR/SSIM、嵌入前后频谱对比、下载按钮。
2. **提取**：上传含水印图 → PNG 元数据自动识别参数，或手动选算法/水印尺寸 → DFT 提示补传原图 → 提取结果 +（若提供原水印）NC/BER。
3. **攻击实验室**：一键复用刚才的嵌入结果或上传 → 选攻击（JPEG 质量 / 缩放比 / 噪声强度滑杆）→ 攻击后图像 + 重提取水印 + 前后指标对比。

**参数回执彩蛋**：嵌入产出的 PNG 将提取所需参数（算法、强度、水印尺寸）写入 PNG tEXt 元数据；提取页拖入后自动填参，普通用户零门槛完成闭环。

### 2.3 视觉系统（DESIGN.md 种子）

- **色彩**（OKLCH）：零彩度或微偏品牌绿色相的真灰白底（明确规避奶油/米黄 AI 默认底色）+ 近黑墨色正文 + 墨绿主色。第 7 幕是全站唯一 Drenched 时刻。
- **字体**：标题衬线 = 思源宋体（Noto Serif SC）× 西文 STIX Two Text（科学期刊字体，呼应主题，不在 impeccable 反射封禁名单）；正文与 UI = 系统无衬线栈。
- **动效**：150–250ms，指数缓出；滚动叙事是唯一大型编排；指标数字滚动计数。
- **图即装饰**：不用装饰性插画，视觉素材全部为真实频谱图、差值图、提取结果。
- 遵循 impeccable 绝对禁令：无渐变文字、无玻璃拟态、无侧边彩条、无 hero-metric 模板、无每节 eyebrow 标签。

## 3. 技术设计

### 3.1 架构与仓库布局

单仓库，webapp 为子目录，后端直接 import `src/watermarking/`（零改动复用）：

```
frequncy_domain_watermarking/
├── src/watermarking/          # 现有算法（不动）
├── webapp/
│   ├── backend/
│   │   ├── app.py             # FastAPI 入口，服务 API + 前端静态文件
│   │   ├── routes.py          # /api/* 路由
│   │   └── services.py        # 薄封装：算法调用、文字→位图渲染、PNG 元数据读写
│   ├── frontend/              # Vite + React + TypeScript + Tailwind
│   │   └── src/               # 滚动叙事、工具区组件、i18n
│   └── scripts/gen_story_assets.py
├── Dockerfile                 # 多阶段：Node 构建前端 → Python 运行时
└── .github/workflows/deploy.yml
```

**无状态**：图片全程内存处理（multipart 进、base64 JSON 出），不落盘、无数据库、不存用户数据（页面明示，作为隐私卖点）。

### 3.2 API

| 端点 | 入参 | 出参 |
|---|---|---|
| `POST /api/embed` | 图片文件、水印（文字或文件）、算法、强度 | 含水印 PNG(b64)、PSNR/SSIM、前后频谱图、参数回执 |
| `POST /api/extract` | 图片文件、算法、水印尺寸、原图(仅 DFT) | 提取水印(b64)、NC/BER（可算时） |
| `POST /api/attack` | 图片文件、攻击类型与参数、提取参数 | 攻击后图像、重提取水印、前后指标 |
| `GET /api/health` | — | 冷启动探测 |

频谱可视化用 numpy + opencv 生成，不引入 matplotlib（镜像瘦身）。

### 3.3 错误处理与免费额度保护

- 输入校验：≤10MB；jpg/png/webp/tiff；处理前最长边缩至 1024px（**产出的含水印图即为该尺寸，界面明示**，保护免费 CPU 也控制水印容量）；水印文字 ≤20 字符；水印图自动二值化。错误文案中英双语。
- 冷启动体验：页面加载即 ping `/api/health`；超 2 秒未响应则工具区显示"服务器唤醒中（约 30 秒）"状态条。
- 限流：slowapi，每 IP 每分钟 20 次处理请求。
- 超时：单请求 30 秒；前端有失败重试提示。

### 3.4 i18n

zh/en 两份 JSON 字典；默认中文，右上角切换；选择持久化到 localStorage；同步更新 `<html lang>`。

### 3.5 测试

- 后端 pytest：三算法嵌入→提取 roundtrip、文字水印渲染（含中文）、PNG 元数据往返、尺寸/类型/超限错误路径。现有算法测试继续运行。
- 前端 vitest：i18n 字典键完整性（zh/en 一一对应）、关键组件；构建通过为 CI 门槛。
- 上线自检：真实浏览器过嵌入→提取→攻击全流程（含手机宽度）。

### 3.6 CI/CD 与部署

- GitHub Actions：PR 跑测试；main 通过后自动推送到 Hugging Face Space（Docker 自动重建）。
- `.dockerignore` 排除仓库内 PDF/zip/数据集等大文件。
- HF Space 免费档：48 小时无访问休眠，唤醒约 30–60 秒（由 3.3 的冷启动体验兜底）；可后续免费绑定自定义域名。
